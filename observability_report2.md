# 可观测性工具分类分析报告

## 第一部分：按数据类型分类分析

### 1.1 Trace（分布式追踪）工具分析
#### 支持工具
- Jaeger
- SigNoz
- SkyWalking
- Langfuse（OpenTelemetry集成）

#### 核心特性分析
| 工具名 | 核心特性                                                                 |
|--------|--------------------------------------------------------------------------|
| Jaeger | - CNCF 毕业项目，标准度高<br>- 兼容 OpenTracing/OpenTelemetry<br>- 支持采样策略（常量/概率/链路/自适应）<br>- 提供服务拓扑图、依赖分析、火焰图<br>- Go 语言开发，性能优秀<br>- 存储支持 Cassandra/Elasticsearch/ClickHouse |
| SigNoz | - OpenTelemetry 原生支持<br>- 单应用内集成 Logs/Traces/Metrics（关联性强）<br>- 提供 Trace 详情页（包含 Span 上下文、关联 Metrics/Logs 跳转）<br>- 支持 Trace 采样、筛选、聚合<br>- UI 友好，适合快速定位问题<br>- 存储支持 ClickHouse |
| SkyWalking | - 开源 APM 系统，Java 生态为主<br>- 支持多种语言探针（Java/Python/Go 等）<br>- 提供 Trace 分析、服务响应时间分布<br>- 支持链路拓扑动态展示<br>- 存储支持 Elasticsearch/InfluxDB/H2/MySQL |
| Langfuse | - LLM 工程化场景下的 Trace 追踪<br>- 深度集成 OpenTelemetry、LangChain、OpenAI SDK<br>- 支持 LLM 请求的详细 Span（Prompt、Completion、Token 统计）<br>- 提供 Trace 与评估（Evals）、指标的关联<br>- 适用于 LLM 应用调试、性能优化 |

#### 同类工具对比
| 对比维度       | Jaeger                                                                 | SigNoz                                                                 | SkyWalking                                                             | Langfuse                                                               |
|----------------|-----------------------------------------------------------------------|------------------------------------------------------------------------|-------------------------------------------------------------------------|------------------------------------------------------------------------|
| **优势**       | - CNCF 标准，生态成熟<br>- 高性能，支持大规模场景<br>- 采样策略丰富   | - 三位一体（Tracing+Metrics+Logs）关联性强<br>- UI 易用<br>- 单应用部署，运维简单 | - 对 Java 生态支持深度最佳<br>- 拓扑分析强大<br>- 轻量级部署选项       | - LLM 场景深度定制<br>- 与 LLM 开发工具链集成好<br>- 支持 Trace 与评估/指标联动 |
| **劣势**       | - 功能相对单一，仅支持 Tracing<br>- 关联其他数据需要手动配置<br>- UI 较简陋 | - 社区规模较 Jaeger/SkyWalking 小<br>- 存储扩展性仍需提升               | - 多语言支持深度不如 Jaeger<br>- OpenTelemetry 兼容性起步较晚          | - 场景局限于 LLM 工程化<br>- 通用分布式系统追踪能力不足                 |
| **性能**       | 高（Go 原生，异步处理）                                               | 中（TypeScript 服务端，ClickHouse 存储）                               | 中高（Java 探针，异步传输）                                             | 中（LLM 请求量通常较小）                                               |
| **易用性**     | 中（需单独部署存储、Agent）                                           | 高（单应用一键部署，开箱即用）                                         | 中（需配置探针、存储）                                                 | 高（与 LLM 开发框架集成，API 简单）                                     |

### 1.2 Metric（指标监控）工具分析
#### 支持工具
- Netdata
- Grafana
- SigNoz
- SkyWalking
- Mlflow（AI 应用指标）
- Langfuse（LLM 应用指标）

#### 核心特性分析
| 工具名 | 核心特性                                                                 |
|--------|--------------------------------------------------------------------------|
| Netdata | - 实时性极强（秒级刷新）<br>- 自动发现系统指标（CPU/内存/磁盘/网络等）<br>- 支持自定义插件采集<br>- 提供可视化仪表盘、告警功能<br>- 轻量级 Agent，资源消耗低<br>- 开源免费，社区活跃 |
| Grafana | - 开源可视化平台，支持多种数据源<br>- 提供丰富的图表类型（折线图/柱状图/饼图/仪表盘等）<br>- 支持告警配置、通知（邮件/ Slack/钉钉等）<br>- 支持面板/仪表盘分享<br>- 生态成熟，插件丰富 |
| SigNoz | - OpenTelemetry 原生指标采集<br>- 支持自定义指标定义<br>- 提供指标与 Trace/Logs 的关联分析<br>- 告警功能（基于 PromQL 语法）<br>- 存储支持 ClickHouse |
| SkyWalking | - 应用性能指标（响应时间/吞吐量/错误率等）<br>- 服务拓扑指标<br>- 支持 JVM/GC 等 JAVA 应用指标<br>- 告警功能（基于指标阈值） |
| Mlflow | - AI 应用指标（模型准确率/召回率/延迟等）<br>- 支持指标与模型版本、实验关联<br>- 提供指标可视化对比<br>- 适用于 AI 模型开发、部署监控 |
| Langfuse | - LLM 应用指标（Token 消耗/响应时间/错误率/评估分数等）<br>- 支持指标与 Prompt 版本、Trace 关联<br>- 提供指标仪表盘<br>- 适用于 LLM 应用成本控制、性能优化 |

## 第二部分：按系统架构分类分析

### 2.1 采集侧工具分析
#### 属于采集侧的工具
- Netdata（Agent）
- Jaeger（Agent/SDK/Collector）
- SigNoz（Agent/SDK/Collector）
- SkyWalking（Agent/Collector）
- Cilium（eBPF 采集）
- RxJava（响应式采集 SDK）

#### 采集方式分析
| 工具名 | 采集方式                                                                 |
|--------|--------------------------------------------------------------------------|
| Netdata | - 轻量级 C 语言 Agent<br>- 自动发现系统指标（无需配置）<br>- 支持自定义插件采集（Python/Go 等）<br>- 实时传输到服务端/本地存储 |
| Jaeger | - SDK（OpenTracing/OpenTelemetry 兼容）：代码侵入式<br>- Agent：Go 语言，轻量，负责转发 Spans<br>- Collector：负责收集、处理、存储 |
| SigNoz | - SDK（OpenTelemetry 原生）：代码侵入式<br>- Agent/Collector：OpenTelemetry Collector 二次封装，支持采集后处理 |
| SkyWalking | - Agent（Java/Python/Go 等）：字节码注入/代码侵入式<br>- Collector：负责收集、处理、存储 |
| Cilium | - eBPF 采集：无代码侵入<br>- 支持网络指标、服务拓扑、Trace 数据采集<br>- 高性能，资源消耗低<br>- 与 Kubernetes 深度集成 |
| RxJava | - 响应式采集 SDK：代码侵入式<br>- 用于 Java 应用内事件/指标采集<br>- 支持异步处理、背压机制 |

#### 同类工具对比
| 对比维度       | Netdata                                                                 | Jaeger                                                                 | SigNoz                                                                 | SkyWalking                                                             | Cilium                                                                 | RxJava                                                                 |
|----------------|-----------------------------------------------------------------------|-----------------------------------------------------------------------|------------------------------------------------------------------------|-------------------------------------------------------------------------|------------------------------------------------------------------------|------------------------------------------------------------------------|
| **侵入性**     | 无（系统级 Agent）                                                     | 中（SDK 代码侵入，Agent 无侵入）                                       | 中（SDK 代码侵入，Agent/Collector 无侵入）                              | 中（Agent 字节码注入/代码侵入）                                         | 无（eBPF 内核级）                                                     | 高（代码侵入，需使用响应式 API）                                       |
| **性能**       | 高（C 语言，资源消耗 <1% CPU）                                         | 高（Go 原生，异步处理）                                               | 中（TypeScript Collector，ClickHouse 存储）                             | 中高（Java 探针，异步传输）                                             | 极高（eBPF 内核级，无用户态开销）                                       | 中（Java 响应式，异步处理）                                             |
| **通用性**     | 高（支持所有主流操作系统）                                             | 高（支持多种语言，OpenTracing/OpenTelemetry 兼容）                     | 高（OpenTelemetry 原生，支持多种语言）                                  | 中（Java 生态为主，其他语言支持深度有限）                               | 中（Kubernetes 场景为主）                                               | 中（Java 生态）                                                       |
| **易用性**     | 高（一键安装，自动发现）                                               | 中（需配置 SDK/Agent/Collector）                                       | 高（单应用部署，OpenTelemetry 自动发现）                               | 中（需配置探针、存储）                                                 | 中（Kubernetes 部署，需了解 eBPF）                                       | 中（需学习响应式编程）                                                 |

## 第三部分：总结

### 各维度最佳工具推荐
| 维度                  | 最佳工具                     | 理由                                                                 |
|-----------------------|------------------------------|----------------------------------------------------------------------|
| **Trace（通用场景）**  | Jaeger                       | CNCF 标准，生态成熟，高性能，支持大规模场景                          |
| **Trace（LLM 场景）**  | Langfuse                     | LLM 工程化深度定制，与开发工具链集成好，支持 Trace 与评估/指标联动    |
| **Metric（通用场景）** | Netdata + Grafana            | Netdata 实时采集，Grafana 强大可视化，组合最佳                        |
| **Metric（AI 场景）**  | Mlflow                       | AI 应用深度定制，支持指标与模型版本/实验关联                          |
| **Metric（LLM 场景）**  | Langfuse                     | LLM 工程化深度定制，支持 Token 统计/评估分数等专属指标                |
| **Log（通用场景）**    | Grafana + Loki/Elasticsearch | Grafana 强大可视化，Loki/Elasticsearch 存储/搜索能力强                |
| **Log（LLM 场景）**    | Langfuse                     | LLM 工程化深度定制，关联 Trace/Prompt 清晰                            |
| **三位一体（Tracing+Metrics+Logs）** | SigNoz                     | 单应用集成，OpenTelemetry 原生支持，关联性强，UI 友好                  |
| **采集（系统级）**    | Netdata                     | 无侵入，自动发现，实时性强，资源消耗低                                |
| **采集（无侵入）**    | Cilium                       | eBPF 内核级采集，无代码侵入，高性能，与 Kubernetes 深度集成            |
| **采集（响应式）**    | RxJava                       | Java 响应式采集，支持异步处理、背压机制                                |
| **服务端（通用）**    | Grafana + Jaeger + Prometheus | Grafana 可视化，Jaeger Tracing，Prometheus Metrics，生态成熟组合        |
| **服务端（AI）**      | Mlflow                       | AI 应用深度定制，支持模型管理、实验对比                                |
| **存储（高性能分析）** | ClickHouse                   | 列存储，实时查询，分析性能强，压缩率高                                |
| **存储（搜索）**      | Elasticsearch                | 全文搜索能力强，适合复杂查询                                          |

### 适用场景总结
| 场景                  | 推荐工具组合                                                                 |
|-----------------------|--------------------------------------------------------------------------|
| **中小团队通用可观测性** | SigNoz（单应用集成）或 Netdata + Grafana + Jaeger（轻量级组合）               |
| **大规模分布式系统**  | Jaeger + Prometheus + Grafana + Cassandra/Elasticsearch + Cilium（无侵入采集） |
| **Java 应用 APM**     | SkyWalking + Elasticsearch/InfluxDB + Grafana                              |
| **LLM 应用工程化**    | Langfuse（OpenTelemetry 集成） + OpenAI SDK/LangChain                      |

### 行业趋势分析
1. **OpenTelemetry 原生支持成为标配**：SigNoz/Langfuse 等新工具均以 OpenTelemetry 为基础，Jaeger/SkyWalking 也在积极兼容，统一数据采集标准成为趋势。
2. **三位一体（Tracing+Metrics+Logs）集成化**：传统分离式工具组合（Prometheus+Grafana+Jaeger+ELK）运维复杂，集成化平台（SigNoz）更受中小团队欢迎。
3. **无侵入采集（eBPF）快速发展**：Cilium 等工具通过 eBPF 内核级采集，避免代码侵入，提升性能，未来将在 Kubernetes 场景广泛应用。
4. **垂直场景深度定制**：Langfuse（LLM）、Mlflow（AI）等工具针对特定场景深度定制，提供专属指标、Trace 追踪和可视化，满足垂直领域需求。
5. **AI 赋能可观测性**：Netdata 等工具已开始引入 AI 技术（异常检测、根因分析），提升问题定位效率，未来 AI 与可观测性的结合将更紧密。
